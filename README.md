# CS 300 Portfolio Reflection

### What was the problem you were solving in the projects for this course?

The projects in this course focused on creating a program that could organize and manage course information for a computer science advising system. The program needed to load course data from a file, store it in an efficient way, allow users to search for courses and their prerequisites, and display the courses in alphanumeric order. The first project focused on comparing the runtime and memory performance of different data structures, while the second project involved building a working C++ program using the data structure that best met the project's requirements.

### How did you approach the problem? Consider why data structures are important to understand.

I approached the problem by first looking at the strengths and weaknesses of vectors, hash tables, and binary search trees. Comparing their Big O performance helped me see how each data structure would affect searching, inserting, and organizing the course data. Understanding data structures is important because the choice you make can have a big impact on how efficient and maintainable a program is. After comparing the options, I chose a binary search tree because it offered a good balance between efficient searching and keeping the courses in sorted order without needing an extra sorting step.

### How did you overcome any roadblocks you encountered while going through the activities or project?

There were several challenges I ran into during the project. One of them was figuring out how to correctly validate prerequisites and making sure the program could handle formatting issues in the input file. I also had some difficulty at first getting the sorting to work properly and ensuring the output was displayed in the correct alphanumeric order. I worked through these issues by testing small sections of the code, carefully reviewing the project requirements, and fixing problems step by step until everything functioned as expected.

### How has your work on this project expanded your approach to designing software and developing programs?

These projects helped me see how important it is to plan before writing code. Writing pseudocode and comparing different data structures helped me think more carefully about how the program should be designed before I started coding. I also learned that choosing the right data structure can have a big impact on how well a program works and how efficient it is. This experience made me want to spend more time planning and looking at different approaches before I start coding.

### How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

Understanding the strengths and weaknesses of each data structure helped me make better decisions instead of just picking one at random. It also helped me see how important efficiency is when working with larger amounts of data. I also learned to write code in a more organized way by breaking it into smaller parts and making it easier to read. This makes it simpler to fix problems and update the program later if anything needs to change.
